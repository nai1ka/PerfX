\chapter{Conclusion}
\label{chap:conclusion}

This thesis studied performance monitoring and regression detection for Android applications after release. The main problem is that existing Android monitoring platforms require developers to define alert conditions by hand for individual metrics, screens, and use cases, and they do not group devices into performance cohorts, so monitoring rules become time-consuming to maintain and a regression that affects only one class of devices can stay hidden in the aggregated metrics.

To solve this, a complete system was designed and built. A mobile SDK collects performance metrics on real user devices with low overhead and sends them to the backend in batches. A Ktor backend stores the data in ClickHouse and PostgreSQL and assigns a device cohort to each record. A detection pipeline then compares two consecutive releases by the relative P95 shift, grouped by metric, screen, and cohort, and informs developers when a new release makes performance worse. The system was evaluated for runtime overhead and through an end-to-end test of its regression detection. These results, together with the answers to the research questions, are presented in Chapter~\ref{chap:eval}.

The main contribution of this work is a complete, open-source system that covers the full pipeline, from on-device metric collection to developer alerts. Its novel element is that the release-to-release comparison is stratified by device performance cohort. Android devices differ widely in capability, so the comparison is run separately within groups of devices of similar capability, and a regression that appears only on weaker devices is still detected instead of being averaged away.
